一、安装Zabbix

a.安装什么版本

标准版 长期支持版： LTS

b.zabbix的安装

​ 1.安装repo源

rpm -ivh https://mirrors.tuna.tsinghua.edu.cn/zabbix/zabbix/4.0/rhel/7/x86_64/zabbix-release-4.0-2.el7.noarch.rpm

​ 2.修改repo中的源为清华源后保存 ​ 将repo.zabbix.com修改为mirrors.tuna.tsinghua.edu.cn/zabbix

​ 3.安装zabbix-server-mysql和zabbix-web-mysql

​ yum -y install zabbix-server-mysql zabbix-web-mysql

​ 4.安装配置数据库mariadb-server,导入初始数据

yum -y install mariadb-server.x86_64
systemctl enable mariadb.service
systemctl start mariadb.service

​ 1). 数据库安全初始化，这里我们不设置密码

​ mysql_secure_installation

​ 2).进入数据库进行设置，输入mysql进入数据库

   mysql> create database zabbix character set utf8 collate utf8_bin;
   mysql> grant all on zabbix.* to zabbix@localhost identified	by '123456';
   mysql> flush privileges;

​ 3).查询zabbix导入sql的目录

   rpm -ql zabbix-server-mysql
   cd /usr/share/doc/zabbix-server-mysql-4.0.21/
   gzip -d create.sql.gz
   mysql -uzabbix -p123456 zabbix < create.sql

​ 5.修改zabbix-server配置文件

vim /etc/zabbix/zabbix_server.conf
DBHost=localhost
DBName=zabbix
DBUser=zabbix
DBPassword=123456

systemctl enable zabbix-server.service
systemctl start zabbix-server.service

​ 6.修改zabbix-web配置文件

systemctl enable httpd
systemctl start httpd

​ 7.修改时区

vim /etc/httpd/conf.d/zabbix.conf
php_value date.timezone Asia/Shanghai
systemctl restart httpd

​ 8.Zabbix访问地址

​ 访问地址：192.168.2.200/zabbix ​ 默认用户名和密码 Admin/zabbix

​ 注：安装完成后的配置文件为

Configuration file "/etc/zabbix/web/zabbix.conf.php" created.

二、添加监控主机

1.在Zabbix-Server主机上安装zabbix-agent

yum -y install zabbix-agent.x86_64
systemctl enable zabbix-agent.service
systemctl start zabbix-agent.service

2.其他主机安装zabbix-agent

1）直接清华源上下载zabbix-agent

wget https://mirrors.tuna.tsinghua.edu.cn/zabbix/zabbix/4.0/rhel/7/x86_64/zabbix-agent-4.0.21-2.el7.x86_64.rpm
rpm -ivh zabbix-agent-4.0.21-2.el7.x86_64.rpm

2)修改Server主机为Zabbix-Server的IP地址

vim /etc/zabbix/zabbix_agentd.conf
   Server=192.168.2.200

systemctl enable zabbix-agent.service
systemctl start zabbix-agent.service

四、自定义监控项

说明：这里以主机登录数为例。

1.在zabbix-agent端,找到UserParameter参数

vim /etc/zabbix/zabbix_agentd.conf
    UserParameter=login_users,who|wc -l
    
systemctl restart zabbix-agent.service

2.在zabbix-server安装测试取值工具

yum -y install zabbix-get.x86_64

3.在zabbix-server上测试取值

zabbix_get -s 192.168.2.100 -k login_users

其中192.168.2.100为zabbix-agent的IP地址，login_users为刚才设置的用户参数

4.回到Zabbix浏览器页面，新建监控项 键值： login_users

五、自定义报警

1.自定义报警信息

  • 【邮件报警】

原始报警信息：

Problem started at {EVENT.TIME} on {EVENT.DATE}
Problem name: {EVENT.NAME}
Host: {HOST.NAME}
Severity: {EVENT.SEVERITY}

Original problem ID: {EVENT.ID}
{TRIGGER.URL}

原始恢复操作内容：

Problem has been resolved at {EVENT.RECOVERY.TIME} on {EVENT.RECOVERY.DATE}
Problem name: {EVENT.NAME}
Host: {HOST.NAME}
Severity: {EVENT.SEVERITY}

Original problem ID: {EVENT.ID}
{TRIGGER.URL}

选择“配置”>“动作”>“操作”来自定义报警信息

![image-20200602204551786](/Users/sakura/Library/Application Support/typora-user-images/image-20200602204551786.png)

在消息内容中输入如下信息：

服务器:{HOST.NAME}发生：{TRIGGER.NAME}故障！
{
  告警主机：{HOST.NAME}
  告警地址：{HOST.IP}
  监控项目：{ITEM.NAME}
  监控取值：{ITEM.LASTVALUE}
  告警等级：{TRIGGER.SEVERITY}
  当前状态：{TRIGGER.STATUS}
  告警信息：{TRIGGER.NAME}
  告警时间：{EVENT.DATE} {EVENT.TIME}
  事件ID：{EVENT.ID}
}

在“恢复操作”选项卡的消息内容中输入如下内容：

服务器:{HOST.NAME}恢复：{TRIGGER.NAME}正常！
{
  告警主机：{HOST.NAME}
  告警地址：{HOST.IP}
  监控项目：{ITEM.NAME}
  监控取值：{ITEM.LASTVALUE}
  告警等级：{TRIGGER.SEVERITY}
  当前状态：{TRIGGER.STATUS}
  告警信息：{TRIGGER.NAME}
  告警时间：{EVENT.DATE} {EVENT.TIME}
  事件ID：{EVENT.ID}
}

相关参数内容可参考：
